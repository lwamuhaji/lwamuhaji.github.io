1. DevOps의 개념

DevOps는 개발(Development)과 운영(Operations) 통합해 빠른 개발 및 배포가 가능한 환경을 제공한다. 이를 통해 주기적인 통합과 자동화를 기반으로 효율적이고 지속 가능한 소프트웨어 개발 및 운영을 목표로 한다.

DevOps의 이점

  • 속도: 빠른 작업 속도로 혁신 가속화, 시장 변화 적응, 비즈니스 성과 향상.
    • 예: 마이크로서비스와 지속적 전달로 빠른 업데이트 릴리스.
  • 신속한 제공: 릴리스 빈도와 속도 개선으로 빠른 제품 혁신 및 개선.
    • 지속적 통합/지속적 전달로 소프트웨어 릴리스 프로세스 자동화.
  • 안정성: 안정적이고 빠른 애플리케이션 및 인프라 업데이트 제공.
    • 모니터링, 로깅으로 실시간 성능 정보 제공.
  • 확장 가능: 자동화와 일관성으로 대규모 및 복잡한 시스템 효율적 관리.
    • 예: 코드형 인프라로 반복 가능하고 효율적인 환경 관리.
  • 협업 강화: DevOps 문화로 팀 간 협업, 주인의식, 책임감 강화.
    • 비효율성 감소, 시간 절약, 환경 고려한 코드 작성.
  • 보안: 자동화된 규정 준수와 구성 관리로 보안 유지하며 DevOps 적용.
    • 예: 코드형 인프라와 코드형 정책으로 규정 준수 관리.

2. DevOps에 사용되는 도구와 기능

CI (Continuous Integration)

  • 프로세스: Plan → Code → Build → Test
  • 툴
    • code 단계
      • Git: 코드의 형상관리
      • Confluence, Slack: 팀 협업 및 지식 공유
      • Jira: 이슈 트래킹 및 프로젝트 관리
    • Build 단계
      • 언어 별로 다양한 빌드 툴들이 있다.
        • Java: Maven, Gradle, sbt
        • Javascript: CRA, Vite
    • Test 단계
      • JUnit: Java 전용 유닛 테스트 프레임워크
      • JMeter: 부하테스트 도구
      • Postman: API 테스트 도구

CD (Continuous Delivery)

  • 프로세스: Release → Depoly → Operate → Monitor
  • 툴
    • Jenkins: 앞 선 도구들(git, maven)과 협업해 CI부터 Depoly까지의 파이프라인을 구성한다.
    • Docker: 프로그램을 구동하는데 필요한 환경을 컨테이너 환경에 격리시켜서 여러 프로그램을 구동하더라도 서로 간섭하는 문제가 생기지 않도록 하는 도구이다.
    • Kubernetes: 컨테이너를 대규모로 관리하는데 특화된 도구
    • Datadog: AWS, Azure, GCP와 통합해 서버 상태 및 데이터베이스, 캐시 등 서비스의 모니터링을 지원한다.

컨테이너 (Docker)

도커 이미지는 컨테이너를 만드는 데 사용되는 읽기 전용(Read-only) 템플릿이다. 컨테이너 실행에 필요한 파일과 설정값 등을 포함하고 있는 Dockerfile을 만든 후 빌드 하면 이미지가 생성된다.

도커 컨테이너는 실행 중인 도커 이미지이다. 하나의 이미지로 여러 개의 컨테이너를 띄울 수 있다.

  • 도커 이미지와 컨테이너의 상태

  • 도커 작동 구조

가상머신 방식에서는, Hypervisor가 Infrastructure의 리소스를 나눠 Guest OS를 생성한다.

도커는 각 컨테이너가 하나의 Host OS를 공유한다. 따라서 불필요한 OS를 생성할 필요가 없어 가상머신보다 가볍다.

3. Cloud Computing

  • IaaS, PaaS, SaaS: 클라우드 서비스 공급자(Cloud Service Provider, 이하 CSP)가 컴퓨팅 자원의 어느 단계까지 고객에게 서비스할 것인가를 기준으로 구분된 모델

IaaS (Infrastructure as a Service)

📄 개요

  • 서비스형 인프라 / Infrastructure-as-a-Service
  • IaaS 서비스 제공 업체(CSP)는 인터넷을 통해 인프라 자원을 제공
  • IaaS 사용자는 클라우드 콘솔 또는 API를 통해 인프라를 제어하고, 애플리케이션, 데이터, 운영체제(O/S), 미들웨어 및 런타임을 직접 처리

🔎 특징

  • 사용자는 IT 자원에 대해 가장 높은 수준의 유연성과 관리 자유도를 가짐

✨ 관련 서비스

  • AWS EC2, NHN Cloud Instance

PaaS (Platform as a Service)

📄 개요

  • 서비스형 플랫폼 / Platform-as-a-Service
  • PaaS 서비스와 마찬가지로 핵심 인프라를 호스팅하고, 운영 체제(O/S)와 미들웨어까지 제공
  • 사용자는 애플리케이션 개발/배포, 서비스 관리를 담당

🔎 특징

  • 사용자는 인프라뿐 아니라 개발에 필요한 플랫폼 관리에 신경 쓰지 않고 웹/앱 개발에 집중할 수 있음

✨ 관련 서비스

  • AWS Elastic Beanstalk, NHN Cloud RDS

SaaS (Software as a Service)

📄 개요

  • 서비스형 소프트웨어 / Software-as-a-Service
  • CSP가 인프라(IaaS)와 미들웨어(PaaS) 뿐 아니라 애플리케이션 형태로 완전한 소프트웨어를 제공
  • 사용자는 웹 브라우저 또는 앱을 통해 소프트웨어에 접속

🔎 특징

  • 사용자는 완전한 솔루션을 이용하기 때문에, 서비스 유지/관리에 가장 효율적이지만 보안 및 성능면에서 제어할 수 있는 부분이 적음

✨ 관련 서비스

  • AWS Marketplace Solution, NHN Cloud Dooray
  • NHN Cloud 마켓플레이스 솔루션
