1. 개요

자바는 1995년 개발된 객체 지향 프로그래밍 언어로, 현재까지도 여러 분야에서 많이 쓰이고 있다. 최근에는 하향 추세를 타고 있지만, 많은 서비스들이 이미 자바로 개발되어 있어 아직 수요가 있다. 특히 한국은 자바 공화국이라고 불릴 정도로 소프트웨어 개발 비율에서 자바가 차지하는 비율이 높다. 따라서 자바를 조사하게 되었다.

2. JVM

자바의 가장 큰 특징은 JVM을 통해 플랫폼에 독립적으로 컴파일되고 실행될 수 있다는 점이다.

자바 프로그램 실행 과정

  1. 프로그래머가 .java 파일을 작성한다.
  1. 자바 컴파일러가 .java 파일을 bytecode(.class)로 변환한다.
  1. JVM이 bytecode를 OS에 맞게 해석해준다.

구조

JVM의 구조는 다음과 같다

  • Class Loader
  • Execution Engine
    • Interpreter
    • JIT Compiler
    • Garbage collector
  • Runtime Data Area
    • Class Area
    • Heap Area
    • PC Register
    • Stack Area
    • Native Method Stack Area
  • JNI

Class Loader

클래스 로더는 Loading → Linking → Initialization 의 세 단계를 거쳐서 .class 파일을 Runtime Data Area에 올린다.

  1. Loading

    class 파일들을 JVM 메모리에 로드한다.

  1. Linking

    class 파일을 사용하기 위해 클래스가 제대로 구성되어 있는지 검증하고 메모리를 할당한다.

  1. Initialization

    클래스 변수들을 적절한 값으로 초기화한다.

Execution Engine

  • Interpreter

클래스 로더가 Runtime Data Area에 올린 바이트코드를 인터프리터가 명령어 단위로 읽어서 실행한다.

  • JIT Compiler

인터프리터 방식은 매번 바이트코드를 기계어로 해석해 실행하므로 속도가 느리다. Just In Time 컴파일러는 프로그램 실행 중에 바이트코드를 실시간으로 읽어서 필요한 부분만 컴파일하고 최적화한다. 기계어로 변환된 코드는 캐시에 저장되기 때문에 재사용시 다시 컴파일할 필요가 없다.

  • Garbage Collector

가비지 컬렉터는 힙 영역에서 더이상 사용되지 않는 메모리를 알아서 해제해준다. 

for (int i = 0; i < 10000; i++) {
  NewObject obj = new NewObject();  
  obj.doSomething();
}

위 코드에서 for 문이 돌고 나면 obj 객체에 접근할 수 있는 방법이 없다. 따라서 해당 객체는 더이상 메모리를 차지하고 있을 필요가 없으며, 이 상태를 unreachable이라고 한다. 가비지컬렉터는 주기적으로 unreachable 상태의 객체들을 정리한다.

Runtime Data Area

  • Class Area

    이 영역은 JVM이 시작되면서 생성되며, 바이트 코드 초기화에 필요한 정보들이 저장된다.

    • 필드: 멤버 변수의 이름, 데이터 타입, 접근 제한자
    • 메소드: 메소드 이름, 매개변수와 리턴타입, 접근 제한자
    • 클래스: Class, Interface 여부, Super Class의 정보
  • Heap Area

    이 영역은 런타임에 동적으로 할당되는 공간이다. new 연산자로 생성되는 객체가 저장되며, 가비지 컬렉터의 대상이 되는 공간이다.

  • Stack Area

    자바 메소드 호출 시 스택프레임이 push되고 메소드가 종료되면 pop된다.

    메소드의 매개변수, 로컬변수, 리턴값 등이 저장된다.

  • PC Register

    스레드가 자바의 메소드를 실행하고 있을 때, 실행 중인 JVM 명령의 주소가 여기에 저장된다.

  • Native Method Stack Area

    자바 이외의 언어로 작성된 네이티브 메소드가 실행될 때, 스택 프레임이 여기에 push 된다.

JNI (Java Native Interface)

자바가 다른 네이티브 언어로 작성된 프로그램을 실행할 수 있도록 하는 인터페이스이다.

Native Method Library에서 라이브러리 및 헤더를 로드해 실행한다.

3. 객체 지향

클래스와 인스턴스

아래 코드는 멤버변수 name과 메소드 eat을 갖는 Animal 클래스를 정의하고, Main 클래스에서 인스턴스화 한 뒤 eat 메소드를 호출하는 코드이다.

public class Animal {
    protected String name; // 멤버변수

    Animal(String name) { // 생성자
        this.name = name;
    }
    
    public void eat() { // 메소드
        System.out.println(this.name + " eats");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal animal = new Animal("Andy"); // Aniaml 클래스 인스턴스화
        animal.eat(); // 인스턴스 메소드 호출
    }
}

상속

상속을 이용하면 부모 클래스의 멤버변수와 메소드를 물려받아 코드를 재사용할 수 있다. 아래 코드는 Animal 클래스를 상속받은 Dog 클래스를 보여준다.

public class Dog extends Animal {
    Dog(String name) {
        super(name); // 부모 생성자 호출
    }

    public void eat() {
        System.out.println("A dog " + this.name + " eats"); // 부모의 name 변수 사용
    }
}

public class Main {
    public static void main(String[] args) {
        Animal dog = new Dog("Andy");
        dog.eat();
    }
}

접근제한자

자바에는 public, protected, default, private 접근제한자가 있다.

  • public: 제한 없음
  • protected: 같은 패키지 또는 자식 클래스에서 접근 가능
  • private: 자기 자신 이외에 접근 불가
  • default: 같은 패키지에서만 접근 가능

예를 들어, 방금 사용했던 예시에서 멤버변수 name의 접근 제한자를 protected에서 private으로 변경하면 Dog 클래스에서 엑세스 관련 에러가 발생한다.

public class Animal {
    private String name; // 접근제한자 변경

    Animal(String name) { 
        this.name = name;
    }
    
    public void eat() {
        System.out.println(this.name + " eats");
    }
}

public class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    public void eat() {
        System.out.println("A dog " + this.name + " eats"); // 에러 발생
    }
}

추상 클래스

추상 클래스는 하나 이상의 추상 메소드를 가지는 클래스를 말한다.

자식 클래스에 따라 부모 메소드의 동작이 달라지는 경우 추상 메소드로 정의하고 구현부는 비워둬서 자식 클래스에서 구현을 강제한다. 실체가 없으므로 추상 클래스는 인스턴스화 할 수 없다.

public abstract class Animal {
    protected String name;

    Animal(String name) { 
        this.name = name;
    }
    
    public abstract void eat(); // 추상 메소드: 바디를 비워둔다.
}

public class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    public void eat() { // 부모의 추상 메소드를 직접 구현
        System.out.println("A dog " + this.name + " eats"); 
    }
}

인터페이스

인터페이스는 그 안에 정의된 메소드가 모두 추상 메소드여야 하고(java 8 부터는 바디가 구현되어 있는 default 메소드도 가질 수 있다) 상속받는 쪽에서 모두 구현해주어야 한다는 점에서 추상 클래스와 닮은 점이 있다.

하지만 큰 범위(부모)와 작은 범위(자식)의 관계가 있는 클래스와 달리 인터페이스는 단지 구현해야할 것들을 모아 놓은 집합이라는 점에서 큰 차이가 있다. 또한 클래스와 달리 다중 상속을 지원한다.

또한 상속 시 확장한다는 뜻의 extends가 아닌 구현한다는 뜻의 implements를 사용한다.

public interface Swimmable {
    void swim();
}

public class Dog extends Animal implements Swimmable {
    Dog(String name) {
        super(name);
    }

    public void eat() { 
        System.out.println("A dog " + this.name + " eats");
    }
    
    public void swim() { // 인터페이스의 추상 메소드를 구현
        System.out.println("A dog " + this.name + " swims");
    }
}
