0. 개요

객체지향의 주요 개념을 설명하면서, 이를 객체지향 언어인 자바와 파이썬 코드로 보여준다. 또한 각 언어의 중요한 구성 요소인 JVM, PVM에 대해 간단하게 설명한다.

1. 객체 지향

클래스와 인스턴스

객체지향 프로그래밍에서 클래스는 여러 객체들의 집합이며, 해당 객체들의 공통된 속성(attribute)과 행동(method)을 정의하고 있다.

아래 속성과 행동을 갖는 Animal을 클래스로 표현하고, 해당 클래스를 인스턴스화 해보자.

  • 속성: name
  • 행동: eat()

Java
public class Animal {
    protected String name; // 멤버변수

    Animal(String name) { // 생성자
        this.name = name;
    }
    
    public void eat() { // 메소드
        System.out.println(this.name + " eats");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal animal = new Animal("Andy"); // Aniaml 클래스 인스턴스화
        animal.eat(); // 인스턴스 메소드 호출
    }
}

Python
class Animal:
    def __init__(self, name):
        self.name = name
    
    def eat(self):
        print(self.name + " eats")

animal = Animal("Andy")

클래스(static) 변수

만약 Animal 클래스로 인해 생성된 인스턴스의 전체 개수를 알고 싶다면 어떻게 해야 할까? 변수를 하나 만들고 생성자 함수가 호출될 때마다 1을 더해주면 될 것이다. 또한 그런 정보를 담고 있는 변수는 인스턴스에 종속되어 있지 않고, 클래스에 종속되어 있어야 한다.

이를 클래스 변수라고 하는데, 자바에서는 앞에 static을 붙여서 선언하며, 파이썬의 경우에는 반대로 아무것도 붙이지 않으면 클래스 변수가 된다.

아래 코드는 Aniaml 클래스 내에 클래스 변수 number를 선언하고, 생성자가 호출될 때마다 1을 더하는 모습을 보여준다.

Java
public class Animal {
		static int number = 0; // 클래스 변수
    protected String name;

    Animal(String name) { 
        this.name = name;
        number++;
    }
    
    public void eat() { 
        System.out.println(this.name + " eats");
    }
}

Python
class Animal:
		number = 0 # 클래스 변수

    def __init__(self, name):
        self.name = name # self가 붙은 변수는 인스턴스 변수이다.
        number += 1
    
    def eat(self):
        print(self.name + " eats")

클래스(static) 메소드

클래스 변수와 동일한 개념이 메소드에도 존재한다. 메소드 변수와 동일하게 클래스 메소드는 인스턴스가 없어도 클래스 자체를 통해 접근할 수 있다. 아래는 해당 클래스가 만든 인스턴스가 몇 개인지를 리턴하는 함수 getNumberOfAnimals()를 각각 자바와 파이썬으로 구현한 것이다.

자바는 static을 붙여서 클래스 메소드를 만든다. 반대로 파이썬은 매개변수 self를 없앰으로써 클래스 메소드를 만들 수 있다. (파이썬의 모든 인스턴스 메소드는 self 매개변수를 갖는다.)

Java
public class Animal {
		static int number = 0; // 클래스 변수
    protected String name;

    Animal(String name) { 
        this.name = name;
        number++;
    }
    
    public void eat() { 
        System.out.println(this.name + " eats");
    }
    
    public static int getNumberOfAnimals() {
        return number;
    }
}

public class Main {
    public static void main(String[] args) {
        Animal animal1 = new Animal("Andy"); 
        Animal animal2 = new Animal("Kitty");
        System.out.println(Animal.getNumberOfAnimals()); // 2
    }
}

Python
class Animal:
		number = 0 # 클래스 변수

    def __init__(self, name):
        self.name = name
        number += 1
    
    def eat(self):
        print(self.name + " eats")
        
    def getNumberOfAnimals(): # 매개변수에 self를 넣지 않는다.
				return Animal.number

animal = Animal("Andy")
animal = Animal("Kitty")
print(getNumberOfAnimals()) # 2

상속

상속을 이용하면 부모 클래스의 멤버변수와 메소드를 물려받아 코드를 재사용할 수 있다. 아래 코드에서 Dog 클래스는 Animal 클래스를 상속받는다. 이후 Dog의 인스턴스는 부모의 eat() 함수를 물려받아 호출한다.

Java
public class Dog extends Animal { // Animal 클래스를 상속받음
    Dog(String name) {
        super(name); // 부모 클래스의 생성자 호출
    }
}

public class Main {
    public static void main(String[] args) {
        Animal dog = new Dog("Andy");
        dog.eat(); // 물려받은 함수 호출
    }
}

자바에서는 extends 키워드를 통해 상속받는다.

Python
class Dog(Animal): # Animal 클래스를 상속받음
    def __init__(self, name):
        super().__init__(name) # 부모 클래스의 생성자 호출
    
dog = Dog("Andy")
dog.eat() # 물려받은 함수 호출

파이썬에서는 ( ) 안에 부모 클래스를 넣어서 상속받는다.

메소드 오버라이딩ㆍ오버로딩

  • 메소드 오버라이딩

메소드 오버라이딩은 단어 뜻 그대로 원래 있던 메소드에 새로운 내용을 덮어씌우는 것이다. 아래 코드는 부모 클래스인 Animal의 eat() 함수를 자식 클래스인 Dog가 새롭게 정의하는 것을 보여준다.

Java
public class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    public void eat() { // 부모 클래스에 정의된 eat을 덮어씌운다.
        System.out.println("A dog named " + this.name + " eats");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal dog = new Dog("Andy");
        dog.eat();
    }
}

// 결과
// A dog named Andy eats

Python
class Dog(Animal): # Animal 클래스를 상속받음
    def __init__(self, name):
        super().__init__(name) # 부모 클래스의 생성자 호출

    def eat(self): # 부모 클래스에 정의된 eat을 덮어씌운다.
        print(f"A dog named {self.name} eats")
    
dog = Dog("Andy")
dog.eat()

# 결과
# A dog named Andy eats

  • 메소드 오버로딩

메소드 오버로딩에서 overload는 과부하라는 의미를 가지고 있다. 같은 이름의 함수가 2개 이상인 경우 함수가 오버로드 상태에 있다고 할 수 있다. 자식의 메소드만 기능하게 되는 오버라이딩과는 다르게 각 함수가 모두 기능한다.

함수의 이름이 같은 경우, 매개변수의 개수나 타입은 달라야 한다. 해당 함수를 호출하면 매개변수의 형태를 통해 어떤 함수를 호출해야 할지가 결정된다.

Java
public class Main {
    static void foo(String a) {
        System.out.println("foo(String a)");
    }
    
    static void foo(int a) {
        System.out.println("foo(int a)");
    }
    
    static void foo(int a, int b) {
        System.out.println("foo(int a, int b)");
    }
    
    public static void main(String[] args) {
        foo("");
        foo(1);
        foo(1, 2);
    }
}

실행 결과

파이썬은 메소드 오버로딩을 지원하지 않는다. 파이썬은 동적 타입 언어라 타입이 런타임에 결정되기 때문이다. 대신 다음과 같이 가변타입을 이용해 메소드 오버로딩과 비슷한 효과를 낼 수 있다.

def foo(**kwargs):
	print(kwargs)
	
foo(a=1)
foo(a=1, b=2)

접근제한자

자바에는 public, protected, default, private 접근제한자가 있다.

  • public: 제한 없음
  • protected: 같은 패키지 또는 자식 클래스에서 접근 가능
  • private: 자기 자신 이외에 접근 불가
  • default: 같은 패키지에서만 접근 가능

예를 들어, Animal 클래스의 멤버변수 name의 접근 제한자를 protected에서 private으로 변경하면 Dog 클래스에서 엑세스 관련 에러가 발생한다.

public class Animal {
    private String name; // protected -> private 으로 접근제한자 변경

    Animal(String name) { 
        this.name = name;
    }
    
    public void eat() {
        System.out.println(this.name + " eats");
    }
}

public class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    public void eat() {
        System.out.println("A dog named " + this.name + " eats"); // 에러 발생
    }
}

파이썬에서는 변수나 함수를 private으로 만들고 싶다면 이름 앞에 _을 2개 연속으로 붙이면 된다.

class Animal:
    def __init__(self, name):
        self.__name = name # __ 을 이름 앞에 붙여서 private으로 만듦

    def eat(self):
        print(self.__name, "eats")

class Dog(Animal):
    def __init__(self, name):
        super().__init__(name)

    def eat(self):
        print(f"A dog named {self.__name} eats")

dog = Dog("Andy")
dog.eat()

# 에러 발생!
# AttributeError: 'Dog' object has no attribute '_Dog__name'

만약 protected를 구현하고 싶은 경우 _를 하나만 붙이면 된다. 하지만 파이썬이 지원하는 형식이 아니라 암묵적으로 약속된 컨벤션일 뿐이라서 private 처럼 에러가 발생하지 않으며, 따라서 강제성이 없다.

추상 클래스

추상 클래스는 하나 이상의 추상 메소드를 가지는 클래스를 말한다.

자식 클래스에 따라 부모 메소드의 동작이 달라지는 경우 추상 메소드로 정의하고 구현부는 비워둬서 자식 클래스에서 구현을 강제한다. 구현부분이 빠져 있기 때문에 추상 클래스는 인스턴스화 할 수 없다.

Java
public abstract class Animal {
    protected String name;

    Animal(String name) { 
        this.name = name;
    }
    
    public abstract void eat(); // 추상 메소드: 바디를 비워둔다.
}

public class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    public void eat() { // 부모의 추상 메소드를 직접 구현
        System.out.println("A dog " + this.name + " eats"); 
    }
}

자바에서는 abstract 키워드를 붙여서 추상 클래스를 정의한다.

Python
from abc import *

class Animal(metaclass=ABCMeta):
    def __init__(self, name):
        self.name = name

    @abstractmethod
    def eat(self):
        pass # 추상 메소드이기 때문에 구현을 생략한다.

class Dog(Animal):
    def __init__(self, name):
        super().__init__(name)

    def eat(self):
        print(f"A dog named {self.name} eats")

dog = Dog("Andy")
dog.eat()

파이썬에서는 abc 모듈을 import 해야 추상 클래스 기능을 사용할 수 있다. abc 모듈 내의 메타 클래스 ABCMeta를 Animal 클래스의 메타 클래스로 지정하면, Animal 클래스는 추상 클래스가 된다.

인터페이스

인터페이스는 그 안에 정의된 메소드가 모두 추상 메소드여야 하고(java 8 부터는 바디가 구현되어 있는 default 메소드도 가질 수 있다) 상속받는 쪽에서 모두 구현해주어야 한다는 점에서 추상 클래스와 닮은 점이 있다.

하지만 큰 범위(부모)와 작은 범위(자식)의 관계가 있는 클래스와 달리 인터페이스는 단지 구현해야할 것들을 모아 놓은 집합이라는 점에서 큰 차이가 있다. 또한 클래스와 달리 다중 상속을 지원한다.

또한 상속 시 확장한다는 뜻의 extends가 아닌 구현한다는 뜻의 implements를 사용한다.

(파이썬은 인터페이스를 따로 지원하지 않는다)

public interface Swimmable {
    void swim();
}

public class Dog extends Animal implements Swimmable {
    Dog(String name) {
        super(name);
    }

    public void eat() { 
        System.out.println("A dog " + this.name + " eats");
    }
    
    public void swim() { // 인터페이스의 추상 메소드를 구현
        System.out.println("A dog " + this.name + " swims");
    }
}

2. JVM

자바의 가장 큰 특징은 가상머신(VM)을 통해 플랫폼에 독립적으로 컴파일되고 실행될 수 있다는 점이다. 아래는 자바 프로그램이 실행되는 과정을 보여준다.

  1. 프로그래머가 .java 파일을 작성한다.
  1. 자바 컴파일러가 .java 파일을 bytecode(.class)로 변환한다.
  1. JVM이 bytecode를 OS에 맞게 해석해준다.

JVM 구조

JVM을 구성하고 있는 요소는 다음과 같다.

  • Class Loader
  • Execution Engine
    • Interpreter
    • JIT Compiler
    • Garbage collector
  • Runtime Data Area
    • Class Area
    • Heap Area
    • PC Register
    • Stack Area
    • Native Method Stack Area
  • JNI

Class Loader

클래스 로더는 Loading → Linking → Initialization 의 세 단계를 거쳐서 .class 파일을 Runtime Data Area에 올린다.

  1. Loading

    class 파일들을 JVM 메모리에 로드한다.

  1. Linking

    class 파일을 사용하기 위해 클래스가 제대로 구성되어 있는지 검증하고 메모리를 할당한다.

  1. Initialization

    클래스 변수들을 적절한 값으로 초기화한다.

Execution Engine

  • Interpreter

클래스 로더가 Runtime Data Area에 올린 바이트코드를 인터프리터가 명령어 단위로 읽어서 실행한다.

  • JIT Compiler

인터프리터 방식은 매번 바이트코드를 기계어로 해석해 실행하므로 속도가 느리다. Just In Time 컴파일러는 프로그램 실행 중에 바이트코드를 실시간으로 읽어서 필요한 부분만 컴파일하고 최적화한다. 기계어로 변환된 코드는 캐시에 저장되기 때문에 재사용시 다시 컴파일할 필요가 없다.

  • Garbage Collector

가비지 컬렉터는 힙 영역에서 더이상 사용되지 않는 메모리를 알아서 해제해준다. 

for (int i = 0; i < 10000; i++) {
  NewObject obj = new NewObject();  
  obj.doSomething();
}

위 코드에서 for 문이 돌고 나면 obj 객체에 접근할 수 있는 방법이 없다. 따라서 해당 객체는 더이상 메모리를 차지하고 있을 필요가 없으며, 이 상태를 unreachable이라고 한다. 가비지컬렉터는 주기적으로 unreachable 상태의 객체들을 정리한다.

Runtime Data Area

  • Class Area

    이 영역은 JVM이 시작되면서 생성되며, 바이트 코드 초기화에 필요한 정보들이 저장된다.

    • 필드: 멤버 변수의 이름, 데이터 타입, 접근 제한자
    • 메소드: 메소드 이름, 매개변수와 리턴타입, 접근 제한자
    • 클래스: Class, Interface 여부, Super Class의 정보
  • Heap Area

    이 영역은 런타임에 동적으로 할당되는 공간이다. new 연산자로 생성되는 객체가 저장되며, 가비지 컬렉터의 대상이 되는 공간이다.

  • Stack Area

    자바 메소드 호출 시 스택프레임이 push되고 메소드가 종료되면 pop된다.

    메소드의 매개변수, 로컬변수, 리턴값 등이 저장된다.

  • PC Register

    스레드가 자바의 메소드를 실행하고 있을 때, 실행 중인 JVM 명령의 주소가 여기에 저장된다.

  • Native Method Stack Area

    자바 이외의 언어로 작성된 네이티브 메소드가 실행될 때, 스택 프레임이 여기에 push 된다.

JNI (Java Native Interface)

자바가 다른 네이티브 언어로 작성된 프로그램을 실행할 수 있도록 하는 인터페이스이다.

Native Method Library에서 라이브러리 및 헤더를 로드해 실행한다.

3. PVM

자바에 JVM이 있다면 파이썬에는 PVM이 있다. 자바와 비슷하게 파이썬 인터프리터는 .py 파일을 바로 실행하지 않고, 바이트코드로 변환한 뒤 실행한다. 이때 생성된 바이트코드는 OS에 독립적이다. 이 PVM이 어떤 언어로 구현되어 있는 지에 따라 구현체가 결정된다. 가장 널리 쓰이는 CPython은 PVM이 C로 구현되어 있으며, Java로 구현된 Jython, 파이썬으로 구현된 PyPy 등이 존재한다.

GIL

파이썬 Global Interpreter Lock은 파이썬 인터프리터에서 동시에 여러 스레드가 실행될 때 발생할 수 있는 문제를 방지하기 위해 사용되는 메커니즘이다. GIL은 멀티스레드 환경에서 한 번에 하나의 스레드만이 파이썬 바이트코드를 실행할 수 있도록 제한한다.

  • GIL의 등장 배경

파이썬은 reference counting을 통해 객체의 메모리를 관리한다. 파이썬에는 자바나 C++와는 달리primitive type이 없고 모든 것이 객체이며, 각 객체마다 참조되는 횟수를 카운팅하고 0이 되는 경우에 메모리에서 해제한다(GC).

import sys
a = []
b = a
sys.getrefcount(a)

# 결과
# []는 a, b, sys.getrefcount()에 의해 참조되고 있으므로 3
3

이때 만약 다수의 스레드에서 refcount를 수정하려고 하면 race condition이 발생할 수 있다. 파이썬은 이 문제를 해결하기 위해 전역으로 Lock을 걸어서 동시에 접근할 수 없도록 했다.

  • GIL 동작 방식

위 그림은 세 개의 스레드가 GIL 환경에서 수행되는 흐름을 나타낸 것이다. 기본적으로 I/O 작업이 발생할 때 GIL이 해제/휙득되며 실행 흐름이 다른 스레드로 넘어가는 것을 볼 수 있다. 따라서 I/O 작업이 주를 이루는 환경에서는 파이썬의 멀티스레드 대체적으로 효율적일 것이다.

만약 I/O 작업이 이뤄지지 않는 경우 GIL을 휙득하고 5ms가 지나면 다른 스레드로 GIL을 넘겨주게 된다. 이런 경우가 빈번하다면 문맥교환 비용이 발생해 오히려 싱글스레드에 비해 성능저하가 일어날 수 있다. 따라서 파이썬으로 멀티스레드 프로그래밍을 하는 경우 GIL을 고려해 의도치 않은 결과를 얻지 않도록 해야한다.
