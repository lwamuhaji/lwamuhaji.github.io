1. 개요

파이썬은 1991년 귀도 반 로섬이 개발한 인터프리터 프로그래밍 언어이다. 절차지향, 객체지향, 함수형 프로그래밍을 모두 지원하는 멀티 패러다임 언어이기도 하다. AI와 빅데이터 처리 열풍에 힘입어 TIOBE 언어 순위 1위에 랭크되었으며, 앞으로도 꾸준한 상승세가 예상되기에 이 언어를 조사 대상으로 선택하였다.

2. PVM

자바에 JVM이 있다면 파이썬에는 PVM이 있다. 자바와 비슷하게 파이썬 인터프리터는 .py 파일을 바로 실행하지 않고, 바이트코드로 변환한 뒤 실행한다. 이때 생성된 바이트코드는 OS에 독립적이다. 이 PVM이 어떤 언어로 구현되어 있는 지에 따라 구현체가 결정된다. 가장 널리 쓰이는 CPython은 PVM이 C로 구현되어 있으며, Java로 구현된 Jython, 파이썬으로 구현된 PyPy 등이 존재한다.

3. GIL

파이썬 Global Interpreter Lock은 파이썬 인터프리터에서 동시에 여러 스레드가 실행될 때 발생할 수 있는 문제를 방지하기 위해 사용되는 메커니즘이다. GIL은 멀티스레드 환경에서 한 번에 하나의 스레드만이 파이썬 바이트코드를 실행할 수 있도록 제한한다.

메모리 관리와 GIL

파이썬은 reference counting을 통해 객체의 메모리를 관리한다. 파이썬에는 자바나 C++와는 달리primitive type이 없고 모든 것이 객체이며, 각 객체마다 참조되는 횟수를 카운팅하고 0이 되는 경우에 메모리에서 해제한다(GC).

import sys
a = []
b = a
sys.getrefcount(a)

# 결과
# []는 a, b, sys.getrefcount()에 의해 참조되고 있으므로 3
3

이때 만약 다수의 스레드에서 refcount를 수정하려고 하면 race condition이 발생할 수 있다. 파이썬은 이 문제를 해결하기 위해 전역으로 Lock을 걸어서 동시에 접근할 수 없도록 했다.

위 그림은 세 개의 스레드가 GIL 환경에서 수행되는 흐름을 나타낸 것이다. 기본적으로 I/O 작업이 발생할 때 GIL이 해제/휙득되며 실행 흐름이 다른 스레드로 넘어가는 것을 볼 수 있다. 따라서 I/O 작업이 주를 이루는 환경에서는 파이썬의 멀티스레드 대체적으로 효율적일 것이다.

만약 I/O 작업이 이뤄지지 않는 경우 GIL을 휙득하고 5ms가 지나면 다른 스레드로 GIL을 넘겨주게 된다. 이런 경우가 빈번하다면 문맥교환 비용이 발생해 오히려 싱글스레드에 비해 성능저하가 일어날 수 있다. 따라서 파이썬으로 멀티스레드 프로그래밍을 하는 경우 GIL을 고려해 의도치 않은 결과를 얻지 않도록 해야한다.

class Animal:

변수와 메소드

파이썬은 간결하고 직관적인 문법으로 유명하다. 아래는 Animal 클래스 안에 클래스 변수 num_of_instances를 정의하고, 생성자 메소드에서 인스턴스 변수 name을 초기화하는 코드이다.

여기서 self는 인스턴스 자기 자신을 의미하며, 객체를 통해 메소드를 호출하면 항상 첫 번째 매개변수 자리에 인스턴스가 넘겨지기 때문에 메소드를 정의할 때 첫 번째 매개변수로 항상 self를 지정해 주어야 한다.

class Animal:
    num_of_instances = 0

    def __init__(self, name):
        self.name = name
    
    def display(self):
        print(self.name)

animal = Animal("강아지")
animal.display()

# 결과
강아지


self를 매개변수로 설정하지 않은 get_num_of_instances 메소드의 경우 객체를 통해 호출하면 에러가 발생한다. 이 경우 클래스를 통해 호출해야 한다.

class Animal:
    num_of_instances = 0

    def __init__(self, name):
        self.name = name
        Animal.num_of_instances += 1
    
    def display(self):
        print(self.name)
    
    def get_num_of_instances():
        return Animal.num_of_instances

animal = Animal("강아지")
print(Animal.get_num_of_instances())

# 결과
1

파이썬의 특별한 메소드들(생성자, 연산자, 소멸자 등)은 __으로 시작해서 __으로 끝난다. 아래는 덧셈 연산자 오버라이딩의 예시 코드이다.

class Animal:
    def __init__(self, name):
        self.name = name
    
    def __add__(self, other):
        return self.name + other.name

dog = Animal("강아지")
cat = Animal("고양이")
print(dog + cat)

# 결과
강아지고양이

상속

파이썬에서 상속은 클래스 이름 바로 옆에 괄호를 열고 상속받을 클래스를 넣음으로써 쉽게 이루어진다. 메소드 오버라이딩 역시 같은 이름의 메소드를 선언하면 간단하게 된다.

class Animal:
    def __init__(self, name):
        self.name = name
    
    def eat(self):
        print(self.name, "eats")

class Dog(Animal): # Animal 클래스 상속받음
    def __init__(self, name):
        super().__init__(name) # 부모 클래스의 생성자 호출

    def eat(self): # 메소드 오버라이딩
        print(f"A dog named {self.name} eats")
    
dog = Dog("요크셔테리어")
dog.eat()

# 결과
A dog named 요크셔테리어 eats

접근제한자

파이썬에서 변수나 함수를 private으로 만들고 싶다면 이름 앞에 _을 2개 연속으로 붙이면 된다.

class Animal:
    def __init__(self, name):
        self.__name = name
    
    def eat(self):
        print(self.__name, "eats")

class Dog(Animal):
    def __init__(self, name):
        super().__init__(name)

    def eat(self):
        print(f"A dog named {self.__name} eats")
    
dog = Dog("요크셔테리어")
dog.eat()

# 에러 발생!
AttributeError: 'Dog' object has no attribute '_Dog__name'

protected의 경우 _를 하나만 붙이면 된다. 하지만 파이썬이 지원하는 형식이 아니라 암묵적으로 약속된 컨벤션일 뿐이라서 private 처럼 에러가 발생한다던지 하는 강제성은 없다.
